관계 대수 
 - 어떤 데이터를 어떻게 찾는지에 대한 처리 절차를 명시하는 절차적 언어 
 
 관계 해석 
 - 어떤 데이트를 찾는지만 명시하는 선언적 언어
 
 관계대수 
 - 원하는 결과를 얻기 위해 수학의 대수와 같은 연산을 이용하여 질의하는 방법을 기술하는 언어
 
 
 관계의 수학적 의미 
   - 카디전 프로덕트 (A X B)
	  1) 각 원소의 순서쌍의 집합 
	  2) 각 기초 집합이 가질 수 있는 값의 범위를 도메인이라고 한다(자료형, 제약조건)
	  3) 릴레이션은 카티전 프로덕트의 부분집합
	  4) 집합에서 사용가능한 연산은 합집합, 교집합, 카디전 프로덕트(순서쌍)
	  5) 데이터베이스에서 릴레이션은 수학적 릴레이션 개념을 현실세계에 적용한 것
	  
	  A = {1, 2, 3}
	  B = {1, 2, 3}
	  A X B = ({1,1}, {1,2}, {1,3}, {2,1}, {2,2}, {2,3}, {3,1}, {3,2}, {3,3})
	  {1,1}
	  
	  seq = {1,2,3,4,5}
	  company_name = { "회사1", "회사2", "회사3" }
	  1, 회사1
	  2 회사2
	  3 회사3
	  
관계대수 연산자
  - 순수 관계 연산 
	 1) 관계 데이터 모델을 위해 고안된 순수 관계 연산에 사용되는 연산자
	 2) 셀렉션, 프로젝션, 조인, 디비전, 개명
         셀렉션 - (WHERE -> 검색 조건)	    
		 프로젝션 - 추출된 속성들,
		 조인  - 릴레이션간 관계(유도 연산 - 셀렉션, 카티전 프로턱트)
		 디비전 - IN,  
		 개명 - AS
		
  - 일반 집합 연산
     1) 수학의 집한 이론에서 차용된 연산자
	 2) 합집합, 차집합(R - S), 교집합, 카티전 프로덕트(R X S)

	
	다른 연산자조합을 통해 유도된 연산자 - 유도 연산자
	예) R, S 교집합 = R - (R - S)
	
	
관계 대수식 
   1) 릴레이션 간 연산을 통해 결과 릴레이션을 찾는 절차를 기술한 언어
   2) 단항 연산자 : 연산자<조건> 릴레이션 
   3) 이항 연산자 : 릴레이션1 연산자 <조건> 릴레이션2
   4) 중복된 투플은 존재하지 않는다. 만약 중복된 투플이 있는 경우 하나만 표시
   
 셀렉션과 프로젝션 - 단항 연산자
셀렉션
   - 릴레이션의 투플을 추출하기 위한 연산 
   - 단항 연산자
   
 프로젝션 
    - 릴레이션의 속성을 추출하기 위한 집합 연산
	- 단항 연산자

집합연산 
	이항연산자
   1) 합집합
   2) 교집합
   3) 차집합 (R - S)
   --- 속성수가 같고 순서도 같아야 하며, 동일한 도메인으로 구성(합병가능 조건)
   
   4) 카티전 프로덕트 (R X S) 
     - 두 릴레이션을 연결시켜 하나로 합칠 때 사용 
     - 속성과 도메인이 동일할 필요는 없음 
     - 두 릴레이션을 무조건 수평으로 합친 결과를 반환하므로 결과에 의미가 없으며 대부분 유용한 자료로 활용하기 힘들다
	 - 셀렉션과 프로젝션 연산을 조합하여 두 릴레이션의 관계에 따른 의미 있는 결과를 찾으면 유용하게 
	 사용할 수 있다 
	 
조인 
  1) 두 렐레이션의 공통 속성을 기준으로 속성 값이 같은 투플을 수평으로 결합하는 연산이다
  2) 두 릴레이션을 카티젼 프로덕트 연산을 한 후 셀렉션 연산을 한 것으로 정의
  3) 두 릴레이션의 조인에 참여하는 속성이 서로 동일한 도메인으로 구성
  4) 조인 연산의 결과는 공통 속성의 속성 값이 동일한 투플만 반환
  
  세타 조인 
     - 조인에 참여하는 두 릴레이션의 속성 값을 비교하여 조건을 만족하는 투플만 반환

  동등 조인(INNER JOIN)
    - 세타조인에서 = 연산자를 사용한 조인
	
  자연조인 
    - 동등 조인에서 조인에 참여한 속성이 두번 나오지 않게 반환
	
  외부조인 
    - 자연 조인 시 조인에 실패한 투플을 모두 보여주되 값이 없는 대응 속성에는 NULL 값을 채워서 반환
	- 왼쪽 외부 조인(LEFT OUTER JOIN - LEFT JOIN )
	- 오른쪽 외부 조인(RIGHT OUTER JOIN - RIGHT JOIN)
	- 완전 외부 조인(FULL OUTER JOIN)
	
	yh_member, yh_cart 
	(회원)            (장바구니)
	회원목록은 전부 확인하고 싶은데, 회원원중에서 장바구니 담겨 있는 상품 데이터 
	
  세미조인 
    - 자연조인을 한 후 두 릴레이션 중 한쪽 릴레이션의 결과만 반환 
	
   디비전 
     - 디비전 연산은 특정 값들을 모두 갖고 있는 투플을 찾을 때 사용한다.
	 속성 IN ('b1') -> s1, s2, s3, s4 ..
