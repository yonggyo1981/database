데이터 정의 - DDL(Data Definition Language) - 데이터 정의어
					- 일관성(자료형, 제약조건)
데이터 변경 - DML(Data Manipulation Language) - 데이터 조작어

데이터베이스 언어
1. 데이터 정의어(DDL) - 자료형, 제약조건... 
2. 데이터 조작어(DML) - 데이터를 조회, 변경, 수정, 삭제
3. 데이터 제어어(DCL) - 데이터의 사용자 권한

3단계 데이터베이스 구조
외부 스키마 - 전체 데이터 중 특정 그룹에서만 필요한 데이터의 구조
개념 스키마 - 전체 데이터에 대한 구조
내부 스키마 - 저장장치에서 실제로 저장되는 방법
				 (자료형, 제약조건 ....)
				 
스키마 - 데이터의 구조(자료형 + 제약조건...)
				 
				 
데이터 독립성 
1) 논리적 데이터 독립성
개념스키마 -> 외부 스키마에 영항 X


2) 물리적 데이터 독립성
내부 스키마(저장장치 관련 - 물리적) -> 개념 스키마 X

관계 데이터 모델 
1. 수학의 집합 이론에 근거
2. 비절차적 언어

SELECT 컬럼....(3) FROM 테이블(1) WHERE 조건(2)


릴레이션 - 행과 열로 구성된 테이블
			- (테이블)
			- 수학의 집합에서 나온 개념
			
			도서번호 = {1,2,3,4,5}
			도서이름 = { 축구의역사,.....}
			도서 가격 = {30000, .....}
			
			카티션 프로덕트
			1, 축구의 역사, 굿스포츠, 30000

관계 
	1. 릴레이션 내에서 생성되는 관계
	2. 릴레이션 간에 생성되는 관계 
		(테이블 - 릴레이션 간의 공통 조건에 따른 곤계)

스키마 - 데이터의 구조(데이터가 저장되는 방식- 자료형, 제약조건)
		 - DDL(CREATE DATABASE, CREATE TABLE ...)
		
		- 속성(컬럼) - 각각 항목
		- 도메인 - 속성이 가질 수 있는 값의 집합 
					 (자료형, 제약조건... )
		- 차수 - 속성의 갯수 
		- 카디날리티 - 행(투플)의 갯수

차수  - 컬럼(열)의 총 갯수 

카디날리티 - 행(투플)의 총 갯수

관계 연산  - 데이터를 검색하는 연산(관계 대수)

	- 제약조건 					--- SQL로 제약 선언 	   - DDL(데이터 정의어)
	- 관계대수                   --- SQL 연산               - DML(데이터 조작어)
	

무결성 제약조건
  - 데이터의 일관성
  - 키(Key) 
		1) 유일하게 식별하는 값 
		2) 특정 투플을 식별할때 사용하는 속성, 속성의 집합
		3) 릴레이션 간의 관계를 맺는 데도 사용한다
			릴레이션 - 테이블
  
		슈퍼키  - 투플을 유일하게 식별할 수 있는 하나의 속성 또는 속성의 집합
		
					고객번호, 주민번호, 이름, 주소, 핸드폰
					
					고객번호  - O 
					주민번호 - O
					이름 - X
					주소 - X 
					나이 - X
					핸드폰 -  O
					이메일 -  O
					
					고객번호 + 주민번호 -> O 
					고객번호 + 주민번호 + 이름 -> O
					고객번호 + 주민번호 + 이름 + 주소 -> O
					고객번호 + 주민번호 + 이름 + 주소 + 핸드폰 -> O
					
					
		후보키 - 투플을 유일하게 식별할 수 있는 속성, 속성의 집합 - 최소 집합
					고객번호 , 주민번호, 휴대폰번호 ... 
					
		복합키 
			주문 
			 고객번호 도서번호 판매가격 주문일자 
			 
			 (고객번호 + 도서번호) -> 키 -> 투플을 식별할 수 있는 속성 
		
		기본키(Primary Key) 
			여러 후보키 중에서 대표로 삼는 키(속성, 속성의 집합)
		
		대리키
			주문상품 릴레이션 
			하나의 주문번호 - 여러 상품이 포함
			
			주문번호, 상품번호, ..... 
			
			주문번호 1 - 1000
					    1  - 1001
						2 - 1000
						2 - 1002 
						
			주문번호 + 상품번호 -> 복합키 -> 후보키
			1-1000
			1-1001
			
			idx - 일련번호 - 품주번호
			(주문번호 + 상품번호)
			
			
		대체키
			- 기본키로 선정되지 않은 후보키
				고객번호  - 기본키
				주민번호 -  대체키(후보키)
		
		
		외래키
			- 다른 릴레이션의 기본키를 참조하는 속성을 말한다.
			yh_member - memNo - 기본키
			yh_cart - yh_member의 memNo 기본키를 참조하는 yh_cart.memNo(외래키)
			- 참조하는 기본키와 동일한 제약조건


무결성 제약조건(일관성, 정확성)
	- 도메인 무결성 제약조건 
	
	- 개체 무결성 제약조건 - 기본키 제약조건 
	- 참조 무결성 제약조건 - 외래키 제약조건

		자식 릴레이션(참조하는)

		주문상품번호 - 기본키 
		
		주문상품 릴레이션
